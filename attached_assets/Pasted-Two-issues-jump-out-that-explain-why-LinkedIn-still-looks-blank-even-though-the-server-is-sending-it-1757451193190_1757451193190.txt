Two issues jump out that explain why LinkedIn still looks blank even though the server is sending it:

1) You hydrate from pre.linkedIn only once at mount

Even though your helper should return linkedIn, I don’t want this to hinge on one key. Also, you never re-hydrate after safeMergeResumeData() runs, so any late updates (or a different localStorage shape) won’t hit the form.

2) You fetch /api/profile/draft and merge, but don’t update the form afterward

If that merge adds or normalizes LinkedIn, your UI never picks it up.

Here’s a surgical patch you can paste that makes the form tolerant and rehydrates after the server merge. It also guards against any casing/shape drift.

B) After the server merge, rehydrate from localStorage and only fill if still empty

In your /api/profile/draft effect, add the rehydrate block after safeMergeResumeData(serverDraft):

useEffect(() => {
  const fetchProfileDraft = async () => {
    try {
      console.log('Fetching profile draft...');
      const resp = await fetch('/api/profile/draft');
      if (resp.ok) {
        const serverDraft = await resp.json();
        console.log('Got server draft:', serverDraft);
        safeMergeResumeData(serverDraft);

        // ⬇️ Rehydrate from the latest localStorage (only if empty)
        try {
          const raw = JSON.parse(localStorage.getItem('resumeData') || 'null');
          const bag = raw?.prefill?.step3 || raw?.data?.prefill?.step3 || {};
          const li = bag.linkedin || bag.linkedIn || bag.linkedinUrl || '';
          const cur = bag.currentJobTitle || bag.currentTitle || '';
          setFormData(f => ({
            ...f,
            linkedin: f.linkedin || li,
            currentJobTitle: f.currentJobTitle || cur
          }));
          console.log('♻️ Rehydrated after server merge:', { li, cur });
        } catch (e) {
          console.warn('Rehydrate after merge failed:', e);
        }
      } else if (resp.status === 204) {
        console.log('No server draft available (204)');
      } else {
        console.warn('Profile draft fetch failed:', resp.status);
      }
    } catch (e) {
      console.warn('No server draft; keeping existing local prefill:', e.message);
    }
  };

  fetchProfileDraft();
}, []);

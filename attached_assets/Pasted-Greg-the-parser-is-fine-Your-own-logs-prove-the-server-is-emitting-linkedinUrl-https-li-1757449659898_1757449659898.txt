## The 2 most likely culprits

### 1) Something in the Step-3 flow **overwrites** `localStorage.resumeData` after parsing

You have a smoking gun in your logs: `GET /api/profile-draft 404` right after parse. Very common pattern:

* Step-3 mounts → tries to fetch `/api/profile-draft`
* It 404s → a “fallback init” runs that sets an **empty** draft and writes it to `localStorage`
* That nukes the fresh `prefill.step3.linkedin*` keys the parser just wrote

### 2) The form isn’t actually **reading** the value it has

Typical variants:

* Input is bound to `form.linkedIn` (capital I) but you set `linkedin` in state.
* You only set `placeholder` instead of `value` (so the real value stays empty).
* You call `useForm({ defaultValues })` *before* prefill arrives and never call `reset()`.
* You read `resumeData.prefill.step3` but your saved shape is `resumeData.data.prefill.step3`.

### A) **Stop overwriting** good data on 404 (front-end)

Wherever you handle `/api/profile-draft`:


Add a **safe merge** helper so nothing wipes prefill:

```js
function safeMergeResumeData(incoming) {
  const key = 'resumeData';
  const existing = JSON.parse(localStorage.getItem(key) || 'null') || {};
  // Deep-ish merge but never erase truthy values with "", null, or undefined
  const merged = structuredClone(existing);

  function merge(dst, src) {
    Object.entries(src || {}).forEach(([k, v]) => {
      if (v && typeof v === 'object' && !Array.isArray(v)) {
        dst[k] = dst[k] && typeof dst[k] === 'object' ? dst[k] : {};
        merge(dst[k], v);
      } else {
        const keep = dst[k];
        const incomingEmpty = v === '' || v == null;
        dst[k] = incomingEmpty ? keep ?? v : v;
      }
    });
  }
  merge(merged, incoming);
  localStorage.setItem(key, JSON.stringify(merged));
}
```

```js
// TEMP GUARD: log any write that would remove linkedin/currentJobTitle
(function guardLS(){
  const _setItem = localStorage.setItem;
  localStorage.setItem = function(k, v) {
    if (k === 'resumeData') {
      const prev = JSON.parse(localStorage.getItem(k) || 'null');
      const next = (() => { try { return JSON.parse(v); } catch { return null; } })();
      const prevS3 = prev?.prefill?.step3 || prev?.data?.prefill?.step3 || {};
      const nextS3 = next?.prefill?.step3 || next?.data?.prefill?.step3 || {};
      const prevLinked = prevS3.linkedin || prevS3.linkedIn || prevS3.linkedinUrl;
      const nextLinked = nextS3.linkedin || nextS3.linkedIn || nextS3.linkedinUrl;
      if (prevLinked && !nextLinked) {
        console.warn('🚨 Blocking write that would drop LinkedIn!', { prevS3, nextS3, stack: new Error().stack });
        return; // block destructive write
      }
    }
    return _setItem.apply(this, arguments);
  };
})();
```
